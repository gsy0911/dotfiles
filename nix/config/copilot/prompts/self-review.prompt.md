# ゴール

対象のブランチにて変更されたファイルを取得し、`copilot-instructions.md`にの規約に沿ってレビューを実施する。


# 手順

1. 変更されたファイルを確認する
2. 変更箇所が他のファイルに与える影響を確認する
3. 以下の観点に基づいてレビューを実施し、問題点や改善案を提示する
4. 変更が 0 件なら「変更が見つかりませんでした」と報告して終了

## レビュー観点

* **機能性**: コードが意図通りに動作するか、エッジケースやエラーハンドリングは適切か
* **可読性・保守性**: 命名規則、コードの構造、複雑さ、重複コードの有無
* **セキュリティ**: 入力値の検証、認証・認可の不備、機密情報の扱い
* **パフォーマンス**: 非効率なアルゴリズムやリソースリークの可能性
* **テスト**: 変更に対するテストが十分か、既存のテストを壊していないか
